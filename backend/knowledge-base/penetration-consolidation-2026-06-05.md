```
# 特征: NAS/跳板机同时连接两个隔离网络

  4. 在内部网络进一步横向移动

# 关键指标:
  - ip link show 显示多个接口
  - 路由表 (ip route) 显示多网段可达

### 模式11: "硬编码Token的HTTP登录绕过"

```
# 特征: 嵌入式设备/IoT使用硬编码或可预测的CSRF token

  1. 分析登录页面JavaScript
  2. 发现token/checktoken为固定值
  3. 发现密码(HASH(明文+随机数))机制
  4. 无需前端即可模拟登录请求

  - Frm_Loginchecktoken = "4lrg7qaiaAll79lUFSW9yQZb" (硬编码)

```

### 模式13: "SQLite数据库存admin哈希 → 直接篡改"

```
# 特征: 嵌入式/安全设备使用SQLite存储用户凭证

  4. 失败后直接UPDATE密码哈希
  5. 用已知密码的哈希覆盖（如"admin"的MD5: 21232f297a57a5a743894a0e4a801fc3）

# 注意: 
  - 部分设备会检查哈希一致性/文件完整性
  - 修改后需要重启服务才能生效
  - 覆盖哈希后，用"admin"密码登录即可

| 10.0.0.0/8内网 | NSG防火墙策略拦截 | 需要配置防火墙规则或找到绕过路径 |

### 6.3 新认知

1. **安全设备其实最不安全** — NSG防火墙的完整配置可以通过挂载磁盘全部读取，用户密码可以直接修改。物理安全 = 网络安全

2. **"内网防火墙"的策略是盲区** — 我们以为NSG拦截了所有外部访问，但实际上它允许了开发机（192.168.1.180）直接访问其管理口（230/231），因为开发机在"家庭网络"白名单外

3. **虚拟化平台是终极跳板** — 拿到Proxmox宿主机root = 拿到所有VM的root（通过磁盘操作），比任何虚拟化逃逸都直接

4. **信息泄露 > 漏洞利用** — 三次渗透的核心突破全都是信息泄露（Everything HTTP、KeePass暴露、磁盘文件系统），而不是0day或复杂漏洞利用

5. **"越渗透越聪明"可行** — 从案例1到案例3，渗透效率在提升：案例1花了全天才完成，案例3在1小时内完成了更复杂的Proxmox+NSG渗透
