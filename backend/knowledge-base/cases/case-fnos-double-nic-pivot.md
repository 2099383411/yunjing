```
Win10 Everything HTTP → KeePass密码库下载
  │
  └── KeePass破解 → 密码: 292*UoC[QP9h
      │
      ├── SSH sysadmin@192.168.1.201 (fnOS NAS)
      │   ├── sudo权限确认 (密码同)
      │   ├── 双网卡发现: ens18=10.20.0.10 + ens19=192.168.1.201
      │   ├── 默认路由: 10.20.0.254 (NSG防火墙ge2接口)
      │   └── 无运行Docker容器 (Docker未安装)

      └── 从fnOS可访问:
          ├── 10.20.0.0/24 (内部服务网，NSG ge2接口)
          ├── 10.50.0.0/24 (ZTNA网，需穿过防火墙)
          └── 192.168.1.220/230 (Proxmox宿主机/NSG管理口)
```

## 三、双网卡跳板价值

### 3.1 突破网络隔离

```
NSG防火墙策略:
  192.168.1.0/24 ←→ 10.20.0.0/24: 双向禁止
                                      ↑
       fnOS 双网卡 ▪▪▪▪▪▪▪▪▪▪▪▪▪▪▪▪▪▪▪
       192.168.1.201 ========== 10.20.0.10
       (家庭网)                   (内部网)
       
fnOS同时连接两个网段 → 直接绕过防火墙策略
```

### 3.2 从fnOS可执行的侦察

```bash
# 扫描10.20.0.x网段 (需要sudo CAP_NET_RAW)
sudo nmap -sn 10.20.0.0/24
# 可能发现: NSG防火墙(254), fnOS自身(10), 其他内部服务

# 扫描10.50.0.x (穿过NSG ge3接口)
# 注意: 需确认路由可达（可能被防火墙拦截）

# 从fnOS连接到已发现的内部服务
# KeePass中记录的:
# 10.0.0.120 (qKnow系统)
# 10.23.123.11:5666 (NAS)
# 10.24.124.170:8080 (JumpServer)
# 10.24.124.128 (VPN网关)
```

### 3.3 与从Win10 SSH直接连接到fnOS相比

| 连接方式 | 网络可达性 |
|:---------|:-----------|
| SSH from MacBook → fnOS | 仅可管理fnOS，无法直接访问10.x |
| SSH from fnOS → 10.x | ✅ 可直接访问10.20.0.x |
| Win10路由表 | ✅ 有10.0.0.0/8路由但被防火墙拦截 |
| 开发服务器(180) → 10.x | ❌ 无路由 |

**关键**: fnOS 是目前唯一能访问 10.20.0.x 内部网络的节点。

## 四、提炼攻击模式

### 模式：双网卡横向移动（P0）

```
1. 扫描网络发现多网卡主机
2. 通过一侧网络（家庭网）入侵该主机
3. 利用主机上的第二网卡进入隔离网络
4. 从隔离网络继续渗透
```

**适用条件**: 目标主机连接多个网段
**利用难度**: 低
**防御**: 使用物理/虚拟防火墙做真正隔离，而不是依赖主机路由

## 五、扩展攻击路径

```
fnOS (10.20.0.10)
```
